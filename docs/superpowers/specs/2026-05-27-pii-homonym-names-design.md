# Подавление одиночной детекции имён-омонимов в PII — дизайн

Дата: 2026-05-27. Проект «Классный час». Связано с backlog #16 (over-redaction).

## Проблема

Коллега добавил в `lib/pii/names.ts` детекцию косвенных падежей имён/фамилий (коммит `e740561`) — нужное улучшение приватности. Побочный эффект: усилились ложные срабатывания на словах-ценностях. Имена-омонимы (`Вера`, `Надежда`, `Любовь`, `Роман`, `Лев`) и их косвенные формы (`Веру`, `Вере`, `Любовью`, `Романа`...) редактятся как ПДн, когда на самом деле это слова-ценности. Критично для сценариев о ценностях/патриотизме — самой частой темы «Классного часа»/«Разговоров о важном».

Подтверждено эмпирически (прогон `detectNames`):
```
"Любовь к Родине — основа патриотизма."  => ["Любовь"]   ← ложно
"Вера в себя помогает..."                => ["Вера"]     ← ложно
"Роман Толстого «Война и мир»."          => ["Роман"]    ← ложно
"Веру и надежду нельзя терять."          => ["Веру"]     ← ложно (новое от e740561)
```

## Корень

В `lib/pii/names.ts` три правила:
1. **Полное ФИО с отчеством** (`FULL_WITH_PATRONYMIC`) — структурное, не смотрит в словарь.
2. **Имя + Фамилия** (`FIRST_SURNAME`, проверяет `FIRST_NAME_FORMS.has(имя)`).
3. **Одиночное имя** — `FIRST_NAME_FORMS.has(слово)` без контекста. **Единственный источник ложных срабатываний на омонимах.**

## Решение (точечное, только правило 3)

- Новый `AMBIGUOUS_NAMES = new Set(['Вера', 'Надежда', 'Любовь', 'Роман', 'Лев'])` — имена-омонимы частотных слов.
- Сборка `AMBIGUOUS_NAME_FORMS` — объединение `buildNameForms()` по этим 5 именам (включая косвенные формы).
- В правиле 3 добавить гард: пропустить матч, если `AMBIGUOUS_NAME_FORMS.has(m[1])`.
- Правила 1 и 2 **не меняются** → имена-омонимы в сильном контексте (с фамилией/отчеством) детектятся как раньше.

### Изменение поведения
```
"Любовь к Родине — основа..."      до: ["Любовь"]         после: []
"Вера в себя помогает..."          до: ["Вера"]           после: []
"Роман Толстого «Война и мир»"     до: ["Роман"]          после: []
"Вера Иванова провела урок"        до: ["Вера Иванова"]   после: ["Вера Иванова"]  (не сломано)
"для Любови Петровны материалы"    детектится              детектится               (не сломано)
"Звонок Петру был важен."          до: ["Петру"]          после: ["Петру"]         (не-омоним цел)
"Анна подготовила классный час"    до: ["Анна"]           после: ["Анна"]          (не-омоним цел)
```

### Осознанный компромисс
Редкое одиночное упоминание без фамилии (напр. «ученица Вера») перестанет редактиться. Допустимо: имя без фамилии слабо идентифицирует личность, а калечение сценариев о ценностях — заметная регрессия качества. Утечки сильных идентификаторов (телефон/email/СНИЛС/ФИО с фамилией) не затронуты.

## Сопутствующее (раз правим файл)

Починить 7 lint-ошибок Biome из коммита `e740561`:
- trailing whitespace в массиве суффиксов `SURNAME` (`lib/pii/names.ts`);
- отсутствующие финальные переводы строк в `lib/pii/names.ts` и тестах `tests/lib/pii/{detect,names,patterns,scenario-scan}.test.ts`.

Лишний реформат не делаем — только то, что флагует Biome.

## Тестирование (TDD)

`tests/lib/pii/names.test.ts`:
- одиночные омонимы НЕ детектятся: `Любовь к Родине` → нет имени; `Вера в себя` → нет; косвенные `Веру`/`Любовью` одиночно → нет;
- омонимы в контексте детектятся: `Вера Иванова` → `Вера Иванова`; `Льва Николаевича` (отчество) → детектится;
- не-омонимы целы: `Анна` → `Анна`; `Петру` → `Петру`.

# SPEC: РоВ — соответствие ТЗ Минпросвещения (дата, группа УМК, цель+задачи, ценности 809)

**Дата:** 2026-05-31 · **Тип:** одна фаза, ветка `feat/rov-tz-compliance` · **Опорный документ:** `materials/TZ_scenario_RoV.md`.

## Цель фазы
Закрыть 4 расхождения между текущей моделью РоВ-сценария и ТЗ Минпросвещения:
1. **Дата проведения** — обязательно понедельник (ТЗ §5.1, §2: 1-й урок понедельника).
2. **Целевая группа** — авто из выбранного класса (1-2 / 3-4 / 5-7 / 8-9 / 10-11 / СПО), показывается учителю и в экспорте.
3. **Цель + задачи** — конвенция «одна ведущая цель + опц. задачи» в существующем `goals[]`.
4. **Иерархия ценностей** — ведущая (enum 17 Указа 809) + 0..3 сопутствующих (enum 17) + опц. «живые» формулировки темы с обязательной привязкой `basedOn` к 17.

Плюс — точечные правки формы `/app/new` для РоВ (подсказка про курс, default 30 мин, hint группы УМК, поле даты).

## Out of scope
- Постразговор как отдельное поле.
- Региональный компонент как структурный слот.
- СПО QR-анкета.
- Целевые ориентиры воспитания (отдельный каталог).
- Медиа-якоря (структурированные video/interaktiv/razdatka).
- Номер занятия 1–34 как редактируемое поле (только derived hint).
- Жёсткая проверка «учебного года» (sep–may).
- Календарь поводов → фильтр по понедельникам.
- Поле «Ведущая ценность» в форме `/app/new` (учитель правит в редакторе).

## Архитектурные решения

### Контент-схема
Все новые поля — `optional` в jsonb `ScenarioContent`. **Без миграции БД.**
- `lessonDate?: string` (`YYYY-MM-DD`)
- `leadingValue?: Value809`
- `secondaryValues?: Value809[]` (max 3)
- `valueFormulations?: { text: string; basedOn: Value809 }[]` (max 8)

Старое `values: string[].optional()` — оставляем для обратной совместимости (старые сценарии).

### Совместимость со старыми сценариями
Старое `values: string[]` → в редакторе показываем как read-only блок «Унаследованный формат» + кнопка «Конвертировать» (создаёт черновик `valueFormulations` с пустым `basedOn`, пользователь дозаполняет). Новые сценарии пишутся только в новые поля.

### Whitelist 17
Каталог `VALUES_809` — точные формулировки Указа Президента РФ от 09.11.2022 № 809 (ред. 04.03.2026). Все `leadingValue`/`secondaryValues`/`valueFormulations.basedOn` валидируются по этому enum на стороне сервера (как `selectPersonalResults`).

### Маппинг направлений → ведущая ценность (fallback)
`DIRECTION_TO_LEADING_VALUE: Record<Direction, Value809>` — короткий статический mapping для добора, если LLM не вернула валидную ведущую:
- Гражданское → «гражданственность»
- Патриотическое → «патриотизм»
- Духовно-нравственное → «высокие нравственные идеалы»
- Эстетическое → «высокие нравственные идеалы»
- Физическое и здоровье → «жизнь»
- Трудовое → «созидательный труд»
- Экологическое → «жизнь»
- Познавательное → «высокие нравственные идеалы»
- Адаптация к изменяющимся условиям → «достоинство»
- Семейные ценности → «крепкая семья»
- Профориентация → «созидательный труд»
- Здоровый образ жизни → «жизнь»

### Дата + номер занятия
- `isMonday(YYYY-MM-DD): boolean` — корневая проверка.
- `rovLessonNumber(date): number | null` — 1-й понедельник сентября учебного года = №1, диапазон 1..34, иначе null. Учебный год: если месяц ≥ 9, отсчёт от сентября того же года; если < 9, отсчёт от сентября предыдущего.
- `formatLessonDateRu(date): string` — «понедельник, 12 сентября 2026».

Поле `lessonDate` — **опциональное**, `superRefine` валидирует только если присутствует и `lessonType='rov'`.

### Промпт (rov.ts)
- `PROMPT_VERSION → v12-rov-tz-2026-05-31`.
- В `buildRovSkeletonMessages` добавляется `[VALUES_809_CATALOG]` (нумерованный список из 17) и инструкция:
  > «Выбери ровно одну `leadingValue` (ДОСЛОВНО из перечня выше) + 0–3 сопутствующих ценности `secondaryValues` (тоже дословно). Опционально, 0–5 живых формулировок темы `valueFormulations`, где каждая `{text, basedOn}` — `text` это живая формулировка вашими словами («дружба», «Родина»), а `basedOn` — ОБЯЗАТЕЛЬНО одна из 17 базовых.»
- `SKELETON_SCHEMA_HINT` дополняется новыми полями.
- Текущее `values` в каркасе оставляем (LLM может игнорировать), но клиент-стороной перестаём ему доверять — переходим на новые поля.

### Форма `/app/new` (только для `lessonType='rov'`)
- Accent-50 панель: «*«Разговоры о важном» — федеральный курс, 1-й урок понедельника, 30 мин, цикл из 34 занятий. Структура: эмоциональный вход с видео → раскрытие смыслов через обсуждение → рефлексия.*»
- Default `durationMin = 30` (поиск `30` в `allowedDurations`, fallback на первое валидное).
- Под `<select id="grade">` — hint курсивом «УМК группа: {rovGroupLabel(grade)}».
- Поле «Дата проведения (понедельник, необязательно)» — `<input type="date">` + JS `onChange`: если не понедельник, снэп на ближайший понедельник + обновление подписи; иначе показ «понедельник, 12 сен. 2026 (занятие №5 цикла)».
- Передаётся в payload как `lessonDate`.

### Редактор
- Card **«Цель и задачи»** (переименование старой «Цели»):
  - первая строка placeholder «Цель занятия (одна ведущая)»;
  - последующие placeholder «Задача (опционально)»;
  - кнопка «+ Добавить задачу» вместо «+ Добавить цель».
- Card **«Формируемые ценности»** (только `lessonType='rov'`):
  - `<select>` «Ведущая ценность» (17 опций, пустая опция = не выбрано);
  - чипы «Сопутствующие ценности» (мультиселект из оставшихся 16, max 3, исключая выбранную ведущую);
  - список «Живые формулировки темы» — пары `[Textarea]` + `[<select> basedOn]`, кнопки «+ Добавить» / «✕»;
  - если у сценария уже есть legacy `values: string[]` — показывать сверху серый блок «Унаследованный формат: ...» + кнопка «Конвертировать в новый формат» (создаёт `valueFormulations` из старых строк с `basedOn` = первое не пустое из `DIRECTION_TO_LEADING_VALUE[direction]`).
- Card **«Дата проведения»** (rov): отображение `formatLessonDateRu(lessonDate)` + `<input type=date>` для правки + «Очистить» кнопка.

### Экспорт (document-model.ts)
- `metaTable` — для type `'rov'`, ДОПОЛНИТЕЛЬНЫЕ строки в шапке (поверх существующих):
  - «Дата проведения» — `formatLessonDateRu(lessonDate)` + ` (занятие №N цикла РоВ)` если `rovLessonNumber !== null`;
  - «Группа РоВ» — `rovGroupLabel(grade)`;
  - «Формируемая ценность (ведущая)» — `leadingValue`;
  - «Сопутствующие ценности» — `secondaryValues.join(', ')` если непустой.
- Под «Цель» — блок «Задачи» с буллетами `goals[1..]` если `goals.length > 1`.
- После целей/задач — блок **«Формулировки ценностей на занятии»** с буллетами «{text} ({basedOn})» если `valueFormulations` непуст.
- Существующий блок «Личностные результаты» остаётся без изменений.

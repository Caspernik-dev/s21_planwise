# PPTX-парсинг в lib/parse — дизайн

Дата: 2026-05-27. Кейс «Классный час», критерий 3 (парсинг загружаемых материалов).

## Цель

Загрузка плана (`/app/plans`) и своего материала (`/app/new`) принимает `.pptx` наравне с PDF/DOCX/TXT. PPTX проходит тот же путь, что и остальные форматы: лимит 5 МБ, проверка magic-bytes, PII-анонимизация, изоляция по `user_id` — без изменений.

## Архитектура

Расширяем существующий `parseFile` в `lib/parse/index.ts` — без новой абстракции. PPTX встаёт рядом с PDF/DOCX/TXT как ещё один `Kind`.

### `lib/parse/index.ts`
- `Kind` += `'pptx'`.
- `detectKind`: распознаёт `.pptx`; текст ошибки неподдерживаемого формата → «Разрешены PDF, DOCX, PPTX, TXT.».
- `checkMagic`: PPTX — OOXML-архив (ZIP, magic `PK` = `0x50 0x4b`), общая проверка с DOCX, но своё сообщение об ошибке («Файл повреждён или не является PPTX.»).
- Новая `parsePptx(buffer)`:
  - `jszip` через dynamic import (как `pdf-parse`/`mammoth` — не тянем в клиентский бандл).
  - Берём только файлы, матчащие `^ppt/slides/slide\d+\.xml$`.
  - Сортируем **численно** по номеру слайда (slide10 после slide2, не лексикографически).
  - Из каждого слайда извлекаем текст из всех `<a:t>…</a:t>`, склеиваем рунты через пробел.
  - Разэкранируем XML-сущности: `&amp;` `&lt;` `&gt;` `&quot;` `&apos;`.
  - Слайды склеиваем через `\n\n`, пустые пропускаем, результат `.trim()`.

### Зависимость
`pnpm add jszip` — делаем прямой. Сейчас jszip есть только транзитивно (через mammoth); опираться на транзитивную зависимость хрупко.

### UI (точки выхода)
- `accept=".pdf,.docx,.pptx,.txt"` в `app/app/plans/upload-form.tsx` и `app/app/new/page.tsx`.
- Тексты-подсказки про форматы в `app/app/plans/page.tsx` и `app/app/new/page.tsx` → добавить PPTX.

### Changelog
Пункт improvement к текущей версии в `lib/changelog.ts` — загрузка теперь принимает PPTX (пользовательски-заметно).

## Тестирование (TDD)

`tests/lib/parse/parse.test.ts` — мок `jszip` (как уже мокаются `pdf-parse`/`mammoth`):
- извлечение текста из `<a:t>`;
- численный порядок слайдов (slide1/slide2/slide10);
- игнор не-слайдовых XML (`ppt/presentation.xml` и т.п.);
- разэкранирование XML-сущностей;
- отклонение PPTX с неверными magic-bytes.

## Вне scope (YAGNI)

Не парсим заметки докладчика, мастер-слайды, таблицы и диаграммы — только видимый текст слайдов. Этого достаточно для роли «материал/план».

# Спека: ФГОС-compliance P3 — Метапредметные УУД + RAG-корпус рабочих программ

**Дата:** 2026-05-31  
**Статус:** Утверждён  
**Приоритет:** Высокий — закрывает GAP по метапредметным результатам в экспорте и улучшает методику для не-РоВ типов  

---

## Контекст и мотивация

ФГОС ООО п. 32.1 требует наличия в рабочей программе курса ВУД планируемых **метапредметных результатов** (УУД). Текущий генератор формирует личностные результаты (каталог в `personal-results.ts`) и предметные результаты, но метапредметные УУД в схеме, промптах и экспорте отсутствуют — это прямое несоответствие ФГОС.

Параллельно, в папке `materials/` накоплено 55 PDF-файлов (рабочие программы ВУД, ПВД, программы кружков), которые не ingested в RAG. Это лишает генератор методической базы для типов `krujok` и `subject_extension`.

Спека закрывает оба пробела:
- **Блок A** — статический каталог УУД + инъекция в промпты + поле в схеме + секция в экспорте + физкультминутка
- **Блок B** — RAG-ingest ~45 рабочих программ из `materials/` с тегированием `lesson_type`

## Блок A: Метапредметные УУД и структурный compliance

### A1. Каталог УУД (`lib/scenario/meta-results.ts`)

Новый файл по аналогии с `personal-results.ts`. Строгий whitelist дословных формулировок ФГОС.

Структура:
```ts
type MetaLevel = 'NOO' | 'OOO' | 'SOO'
type MetaGroup = 'cognitive' | 'communicative' | 'regulatory'
type MetaResults = Record<MetaGroup, string[]>
export const META_CATALOG: Record<MetaLevel, MetaResults>
export function selectMetaResults(input: Partial<MetaResults> | undefined, catalog: MetaResults): MetaResults
```

**Каталог формулировок (дословно из ФГОС):**

НОО (приказ № 286, п. 41.2):
- Познавательные: «Сравнивать объекты, устанавливать основания для сравнения, устанавливать аналогии», «Объединять части объекта (объекты) по определённому признаку», «Определять существенный признак для классификации», «Находить закономерности и противоречия в рассматриваемых фактах», «Выявлять недостаток информации для решения учебной задачи на основе предложенного алгоритма»
- Коммуникативные: «Воспринимать и формулировать суждения, выражать эмоции в соответствии с целями и условиями общения», «Проявлять уважительное отношение к собеседнику, соблюдать правила ведения диалога», «Признавать возможность существования разных точек зрения», «Строить речевое высказывание в соответствии с поставленной задачей»
- Регулятивные: «Планировать действия по решению учебной задачи для получения результата», «Устанавливать причины успеха/неудач учебной деятельности», «Корректировать свои учебные действия для преодоления ошибок»

ООО (приказ № 287, п. 42.2) — формулировки укрупняются:
- Познавательные: «Выявлять и характеризовать существенные признаки объектов», «Устанавливать существенный признак классификации, основания для сравнения», «Выявлять закономерности и противоречия в рассматриваемых явлениях», «Выдвигать гипотезы об их связях и закономерностях», «Самостоятельно выбирать способ решения учебной задачи»
- Коммуникативные: «Воспринимать и формулировать суждения, выражать эмоции в соответствии с целями и условиями общения», «Выражать свою точку зрения в устных и письменных текстах», «Понимать намерения других, проявлять уважительное отношение к собеседнику», «Сопоставлять свои суждения с суждениями других участников диалога»
- Регулятивные: «Выявлять проблемы для решения в жизненных и учебных ситуациях», «Составлять план действий и определять необходимые ресурсы», «Устанавливать причины успеха/неудач учебной деятельности», «Вносить коррективы в деятельность на основе новых обстоятельств»

СОО (приказ № 413, п. 7.2) — формулировки выходят на «сформированность»:
- Познавательные: «Владеть навыками получения информации из источников разных типов», «Самостоятельно формулировать и актуализировать проблему, рассматривать её всесторонне», «Выявлять закономерности и противоречия в рассматриваемых явлениях», «Анализировать полученные в ходе решения задачи результаты»
- Коммуникативные: «Владеть различными способами общения и взаимодействия», «Развёрнуто и логично излагать свою точку зрения», «Понимать и использовать преимущества командной и индивидуальной работы», «Принимать цели совместной деятельности, организовывать и координировать действия по её достижению»
- Регулятивные: «Самостоятельно осуществлять познавательную деятельность», «Владеть способами самоконтроля, самомотивации и рефлексии», «Давать оценку новым ситуациям, вносить коррективы в деятельность», «Принимать решения в условиях неопределённости»

**`selectMetaResults(input, catalog)`:**
- Валидирует каждую строку из `input` против whitelist (trim + normalize пробелов)
- Для каждой группы: если валидных < 1 — добирает первую из каталога
- Обрезает до 3 на группу максимум
- Возвращает `{ cognitive, communicative, regulatory }` с min 1 в каждой

**Тесты (TDD):** пустой input → минимум из каталога; невалидные строки отфильтровываются; валидные сохраняются; переполнение обрезается.

### A2. Поле схемы (`lib/scenario/schema.ts`)

Добавляется в `scenarioContentSchema`:
```ts
metaSubjectResults: z.object({
  cognitive: z.array(z.string().min(1)).max(3).optional(),
  communicative: z.array(z.string().min(1)).max(3).optional(),
  regulatory: z.array(z.string().min(1)).max(3).optional(),
}).optional()
```

В `skeletonSchema` — мягкая версия (LLM может прислать мусор, `selectMetaResults` исправит):
```ts
metaSubjectResults: z.object({
  cognitive: z.array(z.string()).optional(),
  communicative: z.array(z.string()).optional(),
  regulatory: z.array(z.string()).optional(),
}).optional()
```

`optional()` на всех уровнях — старые сценарии в БД валидируются без изменений. **Без миграции** (поле в jsonb).

### A3. Инъекция в промпты

В `buildSkeletonMessages` каждого типа занятия добавляется секция `[META_RESULTS_CATALOG]`:

```
[META_RESULTS_CATALOG]
Выбери 1-2 формулировки ДОСЛОВНО из каждой группы:

Познавательные УУД:
1. {формулировка из каталога}
2. ...

Коммуникативные УУД:
1. ...

Регулятивные УУД:
1. ...

Верни в ключах metaSubjectResults.cognitive / .communicative / .regulatory.
Копируй ДОСЛОВНО, без изменений и сокращений.
```

Секция генерируется функцией `buildMetaCatalogSection(level, direction)` из `lib/scenario/meta-results.ts`.

После `parseSkeleton` в `lib/scenario/stream.ts` — применяется `selectMetaResults`:
```ts
const catalog = getMetaCatalog(gradeToLevel(input.grade))
skeleton.metaSubjectResults = selectMetaResults(skeleton.metaSubjectResults, catalog)
```

Аналогично тому, как `selectPersonalResults` применяется сейчас.

### A4. Экспорт (`lib/export/document-model.ts`)

**Шапка документа** — новая строка в `metaTable`:
```ts
{ label: 'Форма проведения', value: deriveFormLabel(content.format, lessonType) }
```

`deriveFormLabel` маппит: `беседа → 'беседа с элементами дискуссии'`, `игра → 'дидактическая игра'`, `мастер-класс → 'практикум / мастер-класс'`, `дебаты → 'групповая дискуссия / дебаты'`, `проектная сессия → 'проектная деятельность'`, `киноклуб → 'просмотр и обсуждение'`.

**Новый блок после личностных результатов:**
```
## Планируемые метапредметные результаты

**Познавательные УУД:**
• {формулировка 1}
• {формулировка 2}

**Коммуникативные УУД:**
• ...

**Регулятивные УУД:**
• ...
```

Рендерится только при непустом `metaSubjectResults`. Работает в PDF, DOCX и на странице `/s/[token]` (через `ScenarioReadOnly` → `buildScenarioDocument`) автоматически.

### A5. Физкультминутка

**В промптах** для типов `rov` и `event` при `grade <= 4`:
```
Включи в середину занятия двигательную паузу (физкультминутка, 2-3 минуты).
Пример активности: type='task', text='Учитель: — Встаньте. Повторяйте за мной движения...'
```

**В `lib/scenario/quality.ts`** — новый warning в `checkScenario`:
```ts
if (input.grade <= 4 && input.durationMin >= 40) {
  const hasPhysMinute = stages.some(s =>
    s.activities.some(a => /физкульт|двигат|встань|разминк/i.test(a.text))
  )
  if (!hasPhysMinute) warnings.push(
    'Для начальной школы на занятии 40+ мин нормативно требуется физкультминутка (СП 2.4.3648-20 п. 2.10.3)'
  )
}
```

**`PROMPT_VERSION`** → `v10-uud-2026-05-31` для всех типов занятий.

## Блок B: RAG-ingest рабочих программ (`materials/`)

### B1. Маппинг lesson_type

Константа `MATERIAL_TYPE_MAP` в `scripts/ingest-materials.ts`:

| Паттерн имени файла | lesson_type |
|---------------------|-------------|
| `ПВД_разговоры*`, `ПВД_мои горизонты*` | `rov` |
| `ПВД_Биология*`, `ПВД_Физика*`, `ПВД_Химия*`, `pvd_matematika*`, `ПВД_НОО_Основы*` | `subject_extension` |
| `Программа ВУД*`, `Программа_Край*` | `krujok` |
| `*Профориентация*` | `rov` |
| `*Обучение служением*` | `event` |
| Остальные `Рабочая программа курса*`, `РП курса*`, `ПФК*` | `krujok` |

Пропускаются (нормативные, не методические):
- `ФГОС НОО.pdf`, `ФГОС ООО.pdf`, `ФГОС СОО.pdf`
- `ФОП НОО.pdf`, `ФОП ООО.pdf`, `ФОП СОО.pdf`
- `сан-эпид требования.pdf`

### B2. Скрипт `scripts/ingest-materials.ts`

По аналогии с `ingest-razgovor.ts`. Отличия:
- Источник: `materials/*.pdf` (локальные файлы, не HTTP)
- Использует `pdf-parse` (уже в deps) напрямую — без HTTP
- `lesson_type` из `MATERIAL_TYPE_MAP` по имени файла
- `source = 'materials'` в `rag_documents`
- `grade_min/grade_max = null` (программы охватывают разные уровни)
- Идемпотентно через sha256 — повторный запуск не дублирует
- Команда: `pnpm ingest:materials`

### B3. Чанкинг и embedding

Переиспользует существующий пайплайн:
- `lib/rag/chunk.ts` — структурный чанкинг (300–800 токенов)
- `lib/gigachat/embeddings.ts` — GigaChat embeddings 2560d
- Пишет в `rag_documents` / `rag_chunks` (существующие таблицы)

### B4. Изменения в retrieve (`lib/rag/retrieve.ts`)

Добавляется мягкий фильтр по `lesson_type` в SQL-запрос:
```sql
WHERE (chunk_meta->>'lesson_type' = $lessonType OR chunk_meta->>'lesson_type' IS NULL)
```

Аналогично уже существующему мягкому фильтру `direction` (РоВ-чанки с `direction=null` всегда в кандидатах). Materials-корпус тегирован — для генерации кружка подтягиваются программы кружков, не РоВ.

**Без миграции** — `chunk_meta` уже `jsonb`.

## Что НЕ входит в scope

- Редактирование УУД в UI (поле в редакторе) — достаточно того что генерируется и экспортируется
- Whitelist метапредметных и предметных результатов отдельно по предметам — следующая итерация
- Ingest ФГОС/ФОП/СанПиН в RAG — нецелесообразно (нормативные тексты без методики)
- Физкультминутка для ООО/СОО — нормативно не обязательна, только предупреждение

---

## Технические детали

- **Миграций нет** — `metaSubjectResults` в jsonb `content`, `lesson_type` в jsonb `chunk_meta`
- **Тесты:** TDD для `meta-results.ts` (selectMetaResults — 6 кейсов), unit для `deriveFormLabel`, обновление тестов `checkScenario` (физкультминутка)
- **Деплой:** `git pull && docker compose up -d --build` (миграций нет), после деплоя `pnpm ingest:materials` на проде
- **PROMPT_VERSION:** `v10-uud-2026-05-31` во всех типах промптов

## Успех

- Сгенерированный сценарий содержит блок «Планируемые метапредметные результаты» с 3 группами УУД из каталога ФГОС
- Экспортируемый PDF/DOCX содержит строку «Форма проведения» в шапке
- Для 1-4 класса 40+ мин — `checkScenario` выдаёт warning про физкультминутку
- `pnpm ingest:materials` завершается без ошибок; `rag_documents` пополняется ~45 записями с `source='materials'`
- Генерация кружка по робототехнике подтягивает чанки из программ `subject_extension`/`krujok`

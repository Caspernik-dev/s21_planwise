# Классный час — ИИ-генератор сценариев внеурочных занятий

Веб-приложение, которое помогает классному руководителю за минуту получить готовый, проводимый «под ключ» сценарий классного часа, квиза, беседы или мастерской. Генерация опирается на методические материалы (RAG), календарь значимых дат и библиотеку лучших сценариев сообщества. Все персональные данные обезличиваются локально до отправки во внешний LLM.

> Хакатонный кейс №5. Подробная продуктовая спецификация — `docs/superpowers/specs/2026-05-20-klassniy-chas-design.md`.

---

## Возможности

- **Генерация сценария по форме** — тема, направление воспитания, класс (1–11 и СПО), формат, длительность. Двухэтапная потоковая генерация (SSE): сначала каркас занятия, затем детали с реальным текстом-сценарием для учителя.
- **RAG на методичках** — ответ строится с опорой на загруженный корпус (методические рекомендации, «Разговоры о важном», эталонные сценарии). Гибридный поиск: `0.7·косинус + 0.3·BM25`.
- **Локальная анонимизация ПДн** — телефоны, email, ФИО, СНИЛС, паспорт, ИНН, даты рождения, адреса находятся регулярками и словарями **без отправки куда-либо**. Отправка необезличенных данных во внешний сервис требует явного подтверждения пользователя.
- **Загрузка плана воспитательной работы** (PDF/DOCX) — парсинг в список тем с отслеживанием прогресса «закрыто N из M».
- **Структурный редактор** — правка блоков, перестановка этапов и активностей (↑/↓), добавление и удаление этапов и активностей, точечная регенерация одной активности. Без rich-text — контент остаётся строго типизированным.
- **Экспорт** в PDF (с вшитым кириллическим шрифтом PT Sans) и DOCX — полноценный методический бланк.
- **Сообщество** — лайк сценария + опциональный шаринг (с повторным строгим PII-чеком), библиотека с семантическим поиском, «использовать как есть» (создаёт личную копию), пред-рекомендация похожих готовых сценариев перед генерацией.
- **Календарь поводов** — 25 значимых дат учебного года, привязка сценария к дате.
- **Админ-панель статистики** — агрегаты по генерациям, контенту, пользователям и сообществу (доступ только роли `admin`).
- **Rate-limit** на все дорогостоящие операции, whitelist демо-аккаунтов для жюри.

## Технологический стек

| Слой | Решение |
|---|---|
| Фреймворк | Next.js 15 (App Router) + TypeScript, монолит (Route Handlers + Server Actions) |
| UI | Tailwind CSS + собственные shadcn-style примитивы, шрифты Inter + Onest |
| БД | PostgreSQL 16 + pgvector (вектор 1024d) |
| ORM | Drizzle |
| Auth | Auth.js v5 (next-auth beta), credentials + JWT, bcrypt |
| LLM / эмбеддинги | GigaChat (chat: `GigaChat-2-Max`, embeddings: `EmbeddingsGigaR`) — внешний API, без локальных моделей |
| Экспорт | `@react-pdf/renderer` (PDF) + `docx` (DOCX) |
| Тесты | Vitest |
| Линтер/форматтер | Biome |
| Пакетный менеджер | pnpm 9 |
| Деплой | Docker Compose (Next standalone + Postgres), бюджет 2 vCPU / 4 ГБ RAM |

**Принципиальные ограничения:** локальные LLM запрещены (всё через внешний API), интерфейс только на русском, авторизация только email/пароль.

## Пайплайн генерации

1. Форма ввода параметров занятия.
2. Локальная анонимизация ПДн.
3. Пред-рекомендация: поиск похожих готовых сценариев в библиотеке сообщества.
4. RAG-retrieval: подбор релевантных методических фрагментов.
5. Стрим каркаса занятия (GigaChat, вызов 1).
6. Стрим детализации этапов (GigaChat, вызов 2).
7. Валидация (Zod) + автонормализация хронометража + сохранение.
8. Редактирование в блочном редакторе.
9. Лайк + опциональный шаринг в сообщество.
10. Экспорт в PDF / DOCX.

## Быстрый старт

```bash
cp .env.example .env.local
# заполнить как минимум:
#   AUTH_SECRET   — openssl rand -base64 32
#   GIGACHAT_AUTH_KEY — base64(client_id:client_secret) из личного кабинета GigaChat

pnpm install
pnpm db:up          # поднять Postgres + pgvector в Docker
pnpm db:migrate     # применить миграции (включая включение pgvector)
pnpm dev            # http://localhost:3000
```

Зарегистрируйтесь на `/register`, после входа основное приложение доступно по `/app`.

### Назначить администратора

```bash
pnpm set:admin you@example.com   # выдаёт роль admin, открывает /app/admin
```

### Наполнить корпус методичек (RAG) — опционально, требует ключ GigaChat

```bash
pnpm gen:seed         # сгенерировать эталонные сценарии через GigaChat
pnpm ingest:seed      # загрузить их в RAG
pnpm ingest:razgovor  # загрузить «Разговоры о важном» по sitemap
pnpm calibrate        # подобрать SIMILARITY_THRESHOLD на реальных запросах
```

## Production / автозапуск

На проде приложение запускается как `next start` за nginx (proxy на `127.0.0.1:3000`), Postgres — в Docker. Чтобы сервис и БД переживали перезагрузку VPS:

**1. БД** — у контейнера выставлен `restart: unless-stopped` (в `docker-compose.yml`), поэтому он поднимается при старте Docker автоматически.

**2. Приложение** — systemd-unit лежит в репозитории: `deploy/planwise.service`. Установка (требует root):

```bash
sudo cp deploy/planwise.service /etc/systemd/system/planwise.service
sudo systemctl daemon-reload
sudo systemctl enable --now planwise        # запустить и включить автозапуск
sudo systemctl status planwise              # проверить
journalctl -u planwise -f                   # логи (дублируются в /home/nikit/app.log)
```

Перед `enable` убедитесь, что в `/home/nikit/planwise` сделан `pnpm build`. После деплоя новой версии: `pnpm build && sudo systemctl restart planwise`.

> **Важно:** unit использует абсолютный путь к Node из nvm (`/home/nikit/.nvm/versions/node/v24.15.0/bin`). При смене версии Node через nvm обновите путь в `deploy/planwise.service` (строки `Environment=PATH=` и `ExecStart=`).

## Переменные окружения

Полный список — в `.env.example`. Ключевые:

| Переменная | Назначение |
|---|---|
| `DATABASE_URL` | Строка подключения к Postgres (по умолчанию порт `5433`) |
| `AUTH_SECRET` | Секрет Auth.js (`openssl rand -base64 32`) |
| `AUTH_URL` | Базовый URL приложения |
| `GIGACHAT_AUTH_KEY` | base64(client_id:client_secret) |
| `GIGACHAT_SCOPE` | `GIGACHAT_API_PERS` (физлица) |
| `GIGACHAT_MODEL` | Модель чата (`GigaChat-2-Max`) |
| `GIGACHAT_EMBED_MODEL` | Модель эмбеддингов (`EmbeddingsGigaR`) |
| `GIGACHAT_INSECURE_TLS` | `true` только для dev, если в окружении нет корневого сертификата «Минцифры РФ». **Отключает проверку TLS на весь процесс.** Прод — `false` + `NODE_EXTRA_CA_CERTS` |
| `PG_TSV_LANG` | Словарь для full-text (`russian`, fallback `simple`) |
| `SIMILARITY_THRESHOLD` | Порог пред-рекомендации (по умолчанию 0.78) |
| `MAX_GENERATIONS_PER_DAY` | Лимит генераций на пользователя в день |
| `DEMO_USER_EMAILS` | Аккаунты без лимита генераций (для жюри), через запятую |

## Где лежат промпты

Все LLM-промпты собраны в `lib/scenario/`:

| Файл | Что внутри |
|---|---|
| `lib/scenario/prompt.ts` | Основные промпты генерации: `buildSkeletonMessages` (каркас занятия), `buildDetailsMessages` (детализация этапов), `buildMessages` (single-shot). Сюда же подмешиваются RAG-фрагменты и примеры из сообщества. Версия промпта — константа `PROMPT_VERSION`. |
| `lib/scenario/regenerate.ts` | Промпт точечной регенерации одной активности (`buildActivityMessages`). |

Изменение текста промптов — здесь; при правке генерации обновляйте `PROMPT_VERSION` для трассируемости.

## Приватность

- PII-детекция полностью локальна (regex + словари RU-имён), без сетевых вызовов и без проверки контрольных сумм.
- Анонимизация по умолчанию включена. Отмена замены пользователем требует явного подтверждения с текстом о передаче данных во внешний сервис GigaChat и логируется.
- Шаринг сценария в сообщество — opt-in, с повторным строгим PII-чеком уже на содержимом сценария: если что-то осталось — публикация блокируется.
- «Использовать как есть» из библиотеки создаёт личную копию, оригинал не изменяется.
- Изоляция данных по `user_id` на всех пользовательских запросах.
